## Paste this into the new conversation:

```
Build the /dream skill — a new global Claude Code skill for purposive imagination.

## What /dream is (and isn't)

/dream is purposive imagination. Given a topic, question, or scenario, the skill generates specific hypothetical scenarios and simulates how the agent would respond — what it would do, why, what values would guide it.

Distinct from existing skills:
- /play — exploratory, curiosity-first, end-free (follow threads, find unexpected connections)
- /plan-deep — structured decomposition of an already-conceived plan
- /dream — purposive imagination with intent (preparation, baseline generation, pre-registration of interpretations, counterfactual rehearsal)

The flow /dream fits into: DREAM UP → PLAN → EXECUTE. /dream generates the creative material that /plan-deep structures. You can also chain /play → /dream → /plan-deep: play opens the space, dream instantiates specific scenarios, plan-deep structures action.

## Full architectural context (read first)

Read this file for the concept, connections, and open questions:
~/Playful Sincerity/PS Software/Claude Code Entities/ideas/dreaming.md

Key points from that doc:
- Dreaming is distinct from sleep (consolidation). Sleep processes what happened; dreaming simulates what could happen.
- Dreaming connects to Phantom's imagination-first vision — behavioral analog of visual active inference.
- Dreamed responses pre-register the agent's interpretation, creating behavioral baselines useful for paradigm-drift detection later.
- Connection to earned conviction (Convergence Paper): dreams let the agent deliberate more choices without waiting for real situations.

## What to build

A slash command /dream that:

1. Accepts input (topic, question, scenario, or prompts for one)
2. Generates N hypothetical scenarios probing that input from different angles:
   - Different edge cases
   - Value conflicts
   - Novel ecosystems or contexts
   - Counterfactual variations of actual experience
3. For each scenario, simulates the agent's response:
   - What would I do here?
   - Why? What values would guide this?
   - What would I attend to, what would I prioritize?
4. Saves the dreams to files (suggested: <project>/dreams/YYYY-MM-DD-<slug>.md or ~/claude-system/dreams/ for non-project use)
5. Returns a summary the user can use as planning material or as baseline reference data

## Reference existing skills for structure

Look at how /play and /plan-deep are structured:
- ~/.claude/skills/play/
- ~/.claude/skills/plan-deep/

Match those patterns. SKILL.md is the entry point. Keep it simple for v1.

## Output format

The skill should produce output that:
- Is re-usable as input to /plan-deep (if the user wants to plan from the dreams)
- Is citable later as a behavioral baseline (so record date + entity identity + scenario text + response text)
- Keeps scenarios and responses clearly separated (for comparing over time)

## Build scope

V1 minimum viable:
- SKILL.md describing /dream
- Usage examples
- One live test-run showing it produces useful dream output
- Place at ~/.claude/skills/dream/

NOT in v1 scope (for later):
- Automated dream scheduling (cron/heartbeat invocation)
- Entity-architecture integration (that comes later, once the skill pattern is proven)
- Cross-scenario comparison tooling

## Downstream — why this skill matters

Once /dream is proven in use as a global skill, the pattern gets extracted for Claude Code Entities. Specifically:
- Could become a job in CCE's sleep loop (dream during consolidation)
- Could be a separate scheduled dream process
- The scenarios generated by /dream become the behavioral baseline corpus for paradigm-drift detection (see the three-drift-types paper concept in ~/Playful Sincerity/PS Software/Claude Code Entities/papers/three-drift-types.md)

So the skill isn't just a tool — it's the prototype for a load-bearing entity-architecture mechanism.
```
